# 🚀 Travel Ipsum — Travel Agency Management System

Full-stack web application for managing a **travel agency**, built with **Jakarta Servlets, JSP and JPA/EclipseLink**.  
Designed to manage **customers, employees, services, packages and sales** with role-based access, printable invoices and an earnings dashboard.

Built as a **monolithic MVC project** with server-side rendering, vanilla JavaScript and smooth GSAP animations.

## 🗄️ Database Setup

```sql
CREATE DATABASE IF NOT EXISTS agency;
```

The schema is auto-generated by JPA (`persistence.xml` uses `create` mode).  
No manual table creation is needed.

**Database configuration** (`src/conf/persistence.xml`):

```properties
# Update these values according to your local MySQL configuration.

jakarta.persistence.jdbc.url=jdbc:mysql://localhost:3306/agency?serverTimezone=UTC
jakarta.persistence.jdbc.user=root
jakarta.persistence.jdbc.password=

# Schema generation: create (development), none (production)
eclipselink.ddl-generation=create
```

## 🚀 Installation & Setup

```bash
# Clone the repository
git clone https://github.com/CristianCaraballo/TravelAgency.git
```

### NetBeans (recommended)

1. Open the project in **NetBeans**
2. Make sure **Tomcat 10+** is configured in Services → Servers
3. Update MySQL credentials in `src/conf/persistence.xml`
4. Right-click project → **Clean and Build**
5. Right-click project → **Run** (deploys to Tomcat)

### Manual WAR deployment

```bash
# The WAR is generated at:
dist/TourismAgency.war

# Copy it to Tomcat's webapps directory:
# e.g. C:\apache-tomcat-10.x\webapps\

# Start Tomcat and navigate to:
http://localhost:8080/TourismAgency
```

A running **MySQL instance** is required.

## 🔎 Servlets / Endpoints

| Servlet | URL | Methods | Description |
|---------|-----|---------|-------------|
| `SvLogin` | `/SvLogin` | POST | Login / Logout |
| `SvIndex` | `/SvIndex` | GET | Redirect to new-sale page |
| `SvCustomer` | `/SvCustomer` | GET, POST | Customer CRUD + search by DNI |
| `SvEmployee` | `/SvEmployee` | GET, POST | Employee CRUD (admin only) |
| `SvService` | `/SvService` | GET, POST | Service CRUD (admin only) |
| `SvPackage` | `/SvPackage` | GET, POST | Package CRUD (admin only) |
| `SvSale` | `/SvSale` | GET, POST | Sale CRUD + invoice generation |
| `SvEarns` | `/SvEarns` | GET, POST | Earnings dashboard (admin only) |

## ✨ Features

- 🔐 **Role-based authentication** — Sellers and Admins with different access levels
- 👥 **Customer management** — Full CRUD with DNI search
- 🧑‍💼 **Employee management** — CRUD with integrated user accounts
- 🛎️ **Service management** — Individual travel services CRUD
- 📦 **Package management** — Bundle services with **10% automatic discount**
- 💵 **Sales management** — Create, edit, delete sales with dynamic pricing
- 🧾 **Printable invoice** — Real-time bill preview with payment adjustments
- 📊 **Earnings dashboard** — Filter by today, month, year, or custom range
- 🗑️ **Soft-delete** — Records are deactivated, not permanently removed
- ⚡ **Smooth animations** — GSAP-powered sidebar and hamburger menu
- 📱 **Responsive sidebar** — Collapsible navigation with role-based menu items

## ⚠️ Note

This project uses **Jakarta Servlets** (`jakarta.servlet` namespace).  
It **only works with Apache Tomcat 10+** — earlier versions use the `javax.servlet` namespace and are not compatible.

The database schema is auto-generated by EclipseLink in `create` mode.  
For production, change to `none` and manage migrations manually.
